# Infrasity Sales Sync тАФ Setup Guide
**Auto-syncs your Google Sheet тЖТ salesData.js every day at 4:00 AM IST**

---

## How It Works

```
Google Sheet (you update)
        тЖУ (every day 4am IST)
    sync.js runs
        тЖУ
Reads all sheet tabs тЖТ parses data тЖТ writes salesData.js
        тЖУ
Dashboard loads fresh data automatically
```

## Option 1: GitHub Actions (Recommended тАФ Zero Infrastructure)

This runs the sync in the cloud for free, no server needed.

### Step 1: Create a Google Service Account

1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. Create a new project (or use existing)
3. Enable **Google Sheets API**: APIs & Services тЖТ Enable APIs тЖТ search "Google Sheets API" тЖТ Enable
4. Go to **IAM & Admin тЖТ Service Accounts тЖТ Create Service Account**
5. Name it `infrasity-sync`, click Create
6. Skip role assignment, click Done
7. Click the service account тЖТ **Keys тЖТ Add Key тЖТ Create new key тЖТ JSON**
8. Download the JSON file тАФ this is your `service-account.json`

### Step 2: Share Your Sheet with the Service Account

1. Open the JSON file, copy the `client_email` value (looks like `infrasity-sync@your-project.iam.gserviceaccount.com`)
2. Open your Google Sheet
3. Click **Share**
4. Paste the service account email
5. Set permission to **Viewer**
6. Uncheck "Notify people" тЖТ Click Share

### Step 3: Set Up GitHub Repository

1. Create a GitHub repo (e.g. `infrasity-sales-dashboard`)
2. Structure it like this:
   ```
   infrasity-sales-dashboard/
   тФЬтФАтФА dashboard/
   тФВ   тФЬтФАтФА index.html          тЖР your dashboard file
   тФВ   тФФтФАтФА data/
   тФВ       тФФтФАтФА salesData.js    тЖР auto-generated by sync
   тФЬтФАтФА infrasity-sync/
   тФВ   тФЬтФАтФА package.json
   тФВ   тФЬтФАтФА scripts/
   тФВ   тФВ   тФЬтФАтФА sync.js
   тФВ   тФВ   тФФтФАтФА scheduler.js
   тФВ   тФФтФАтФА .github/
   тФВ       тФФтФАтФА workflows/
   тФВ           тФФтФАтФА sync.yml
   тФФтФАтФА README.md
   ```
3. Copy all files from `infrasity-sync/` folder into the repo
4. Push to GitHub

### Step 4: Add Secret to GitHub

1. Go to your repo тЖТ **Settings тЖТ Secrets and variables тЖТ Actions**
2. Click **New repository secret**
3. Name: `GOOGLE_SERVICE_ACCOUNT_JSON`
4. Value: paste the **entire contents** of your `service-account.json` file
5. Click Add secret

### Step 5: Enable GitHub Pages (to publish the dashboard)

1. Go to repo тЖТ **Settings тЖТ Pages**
2. Source: Deploy from a branch
3. Branch: `main`, folder: `/dashboard`
4. Your dashboard will be live at `https://yourusername.github.io/infrasity-sales-dashboard`

### Step 6: Test It

1. Go to **Actions tab** in your GitHub repo
2. Click "Sync Infrasity Sales Data" workflow
3. Click **Run workflow** тЖТ Run workflow
4. Watch the logs тАФ should complete in ~30 seconds
5. Check that `dashboard/data/salesData.js` was updated in the repo

From now on it runs automatically every day at 4:00 AM IST. тЬУ

## Option 2: Run Locally (on your Mac/PC)

Use this if you want to run it on your own machine.

```bash
# 1. Install Node.js 18+ from nodejs.org

# 2. Clone or download this folder
cd infrasity-sync

# 3. Install dependencies
npm install

# 4. Put your service-account.json in config/
# (copy it from the download in Step 1 above)

# 5. Run once manually
npm run sync

# 6. Run as a daily scheduler (keeps running in background)
npm start

# 7. To keep it running after you close terminal, use PM2:
npm install -g pm2
pm2 start scripts/scheduler.js --name infrasity-sync
pm2 startup    # makes it restart on reboot
pm2 save
```

## Troubleshooting

| Error | Fix |
|-------|-----|
| `No credentials found` | Put service-account.json in config/ or set env var |
| `403 Forbidden` | Share the sheet with the service account email |
| `Tab not found` | Check SHEET_STRUCTURE.md for expected tab names |
| `salesData.js not updated` | Check logs/ folder for error details |
| GitHub Actions not running | Check repo Settings тЖТ Actions тЖТ Allow all actions |
